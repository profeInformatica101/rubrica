¿Qué son las ACL?

Las Access Control Lists (ACL) proporcionan un mecanismo de permisos adicional y más flexible en sistemas Linux. Permiten asignar permisos específicos a cualquier usuario o grupo, complementando los permisos estándar basados en el propietario, el grupo y otros.

Tipos de ACL

Existen diferentes tipos de ACL en función del sistema de archivos:

  • POSIX ACL: Es el tipo de ACL estándar en la mayoría de distribuciones Linux. Compatible con sistemas de archivos como ext4.
  • NFSv4 ACL: Ofrece más granularidad y se utiliza en sistemas que implementan NFS (Network File System).

Comandos Básicos

  • Añadir permisos para un usuario: 
    setfacl -m u:usuario:permisos /ruta/al/archivo
  • Añadir permisos para un grupo: 
    setfacl -m g:grupo:permisos /ruta/al/archivo
  • Eliminar ACL: 
    setfacl -b /ruta/al/archivo
  • Mostrar ACL existentes: 
    getfacl /ruta/al/archivo

Casos de Uso Prácticos

Las ACL son ideales para situaciones donde los permisos estándar no son suficientes. Aquí algunos ejemplos:

  • Colaboración en proyectos: Configurar un directorio compartido donde cada miembro del equipo tenga permisos específicos.
  • Seguridad de datos: Restringir el acceso a archivos confidenciales otorgando permisos solo a usuarios específicos.
  • Aislamiento de usuarios: Crear entornos aislados para diferentes grupos de trabajo, evitando conflictos.

Consideraciones de Seguridad

  • Configuración correcta: Una configuración incorrecta de ACL puede exponer datos sensibles.
  • Auditoría: Usa herramientas como auditd para monitorear cambios en las ACL.
  • Mejores prácticas: Revisa regularmente las configuraciones de ACL para evitar permisos innecesarios.

Preguntas Frecuentes

Los permisos estándar se aplican al propietario, el grupo y otros, mientras que las ACL permiten asignar permisos específicos a cualquier usuario o grupo, ofreciendo mayor flexibilidad.

Usa el comando tune2fs -l /dev/sdX y busca la opción Default mount options. Si aparece acl, el sistema lo soporta.
