Tarea: Aprendiendo los Comandos Básicos de chage

Objetivo

Familiarizarse con el comando chage para gestionar políticas de contraseñas en Linux y comprender cómo aplicarlas en un entorno práctico.

Parte 1: Exploración de chage

  1. Consulta las políticas actuales de un usuario:

    sudo chage -l testuser

    Preguntas:

    • ¿Cuándo caduca la contraseña de testuser?
    • ¿Cuál es el número de días de advertencia antes de la caducidad?

Parte 2: Configuración de Políticas de Contraseñas

  1. Configura las siguientes políticas para el usuario testuser:

    • La contraseña debe caducar después de 60 días.
    • El usuario debe recibir una advertencia 10 días antes de la caducidad.
    • El usuario no puede cambiar la contraseña más de una vez al día.

    Comandos:

    sudo chage -M 60 testuser
sudo chage -W 10 testuser
sudo chage -m 1 testuser

    Preguntas:

    • ¿Qué comando se usó para configurar la advertencia de caducidad?
    • ¿Qué significan las opciones -M y -m en chage?

Parte 3: Obligar a Cambiar la Contraseña

  1. Obliga al usuario testuser a cambiar su contraseña en el próximo inicio de sesión:

    sudo chage -d 0 testuser

    Preguntas:

    • ¿Qué mensaje se muestra al intentar iniciar sesión con testuser?
    • ¿Qué sucede si el usuario no cambia la contraseña al iniciar sesión?

Parte 4: Configuración Avanzada

  1. Establece una fecha de expiración para la cuenta de testuser:

    sudo chage -E $(date -d "+90 days" +"%Y-%m-%d") testuser

    Preguntas:

    • ¿Qué comando muestra la fecha de expiración de la cuenta?
    • ¿Qué sucede cuando se alcanza la fecha de expiración configurada?

Parte 5: Resumen y Verificación

Verifica la configuración completa de testuser:

sudo chage -l testuser
